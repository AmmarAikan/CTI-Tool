Project Name:
AI-Based Cyber Threat Intelligence Tool / CTI-Tool

Part 0: DNRTI NER Model Training Pipeline

Purpose of Part 0
-----------------
The purpose of this part is to train a Named Entity Recognition (NER) model on the DNRTI dataset in order to extract cybersecurity entities from threat intelligence text.

The target entities include examples such as:
- Threat Actor
- Malware / Tool
- Organization
- Location
- Attack Method
- Security Team
- Exploit
- Objective
- Time

Dataset Used
------------
Dataset name:
DNRTI

Dataset location inside the project:
ml/datasets/dnrti/train.txt
ml/datasets/dnrti/valid.txt
ml/datasets/dnrti/test.txt

Dataset format:
The dataset uses BIO tagging. Each line contains a token and its BIO label.

Simplified example:
APT28      B-HackOrg
used       O
X-Agent    B-Tool
malware    I-Tool
Ukraine    B-Area

BIO meaning:
B = Beginning of an entity
I = Inside or continuation of an entity
O = Outside any entity

Verified dataset split sizes:
train = 5251 sentences
valid = 662 sentences
test  = 664 sentences

Model 1: sklearn Baseline
-------------------------
Idea:
The sklearn model is a lightweight local baseline. It does not require downloading a pretrained model and does not require a GPU.
It converts each token into handcrafted features, such as:
- word shape
- digits
- hyphen existence
- uppercase/lowercase patterns
- previous and next token context

Then it trains a classifier to predict the BIO label for each token.

Training script:
ml/training/train_ner_sklearn.py

Saved model:

BERT Output Issue Before the Final Fix
--------------------------------------
After training, BERT worked but some outputs contained WordPiece fragments, such as:
AP
##T
X - Agent ma

Example before the fix:
[
 {'type': 'threat_actor', 'value': 'AP', 'source': 'dnrti_bert_ner'},
 {'type': 'threat_actor', 'value': '##T', 'source': 'dnrti_bert_ner'},
 {'type': 'tool_or_malware', 'value': 'X - Agent ma', 'source': 'dnrti_bert_ner'}
]

Reason:
BERT uses WordPiece tokenization. It may split words into subword pieces, for example:
APT28 -> AP + ##T + ##28

Final Update in ner_extractor.py
--------------------------------
Updated file:
backend/app/pipeline/extraction/ner_extractor.py

Purpose of the update:
Improve the BERT extraction output by:
- using start/end offsets from the original text
- expanding boundaries when they are inside a word
- cleaning WordPiece markers such as ##
- removing extra spaces around hyphens, for example converting X - Agent into X-Agent
- preserving the same output keys:
  type, value, confidence, source

How NERExtractor Works Now
--------------------------
When NERExtractor is initialized:
1. It first tries to load BERT from:
   ml/models/dnrti_bert_ner
2. If config.json and the model files exist, it uses BERT.
3. If BERT is missing or fails to load, it falls back to sklearn:
   ml/models/dnrti_sklearn_ner/model.joblib

This makes the system flexible:
- If BERT exists: use transformer.
- If BERT does not exist: use sklearn baseline.
