معنى BIO:
B = بداية كيان

O = خارج أي كيان

المشكلة التي واجهتنا:

مشكلة مخرجات BERT قبل التعديل
------------------------------
بعد التدريب، كان BERT يعمل لكن المخرجات كانت تحتوي على WordPiece tokens مثل:
AP
##T
X - Agent ma

مثال قبل التعديل:
[
 {'type': 'threat_actor', 'value': 'AP', 'source': 'dnrti_bert_ner'},
 {'type': 'threat_actor', 'value': '##T', 'source': 'dnrti_bert_ner'},
 {'type': 'tool_or_malware', 'value': 'X - Agent ma', 'source': 'dnrti_bert_ner'}
]

سبب المشكلة:
BERT يستخدم WordPiece Tokenization، لذلك قد يقسم الكلمة إلى أجزاء مثل:
APT28 -> AP + ##T + ##28

التعديل النهائي في ner_extractor.py
------------------------------------
تم تعديل الملف:
backend/app/pipeline/extraction/ner_extractor.py

هدف التعديل:
تحسين دالة استخراج مخرجات BERT بحيث:
- تستخدم start/end offsets من النص الأصلي.
- توسع الحدود إذا كانت داخل الكلمة.
- تنظف رموز WordPiece مثل ##.
- تزيل المسافات الزائدة حول الشرطة مثل X - Agent وتحولها إلى X-Agent.
- تحافظ على نفس شكل المخرجات:
  type, value, confidence, source

أمر الاختبار النهائي بعد التعديل:
python -c "from backend.app.pipeline.extraction.ner_extractor import NERExtractor; e=NERExtractor(); print(e.backend); print(e.extract_entities('APT28 used X-Agent malware to target government organizations in Ukraine through spear phishing attack.'))"
